Credential References

Credentials

Dispatch resolves endpoint credentials through named references. References are safe to log; resolved passwords are kept in memory only and are never serialized.

Where Credential Data Lives

File or optionPurpose
C:\ProgramData\Dispatch\config.ymlDefines credential references, provider types, usernames, store paths, Key Vault URIs, secret names, and auth modes.
hosts.ymlMay select a credential reference for defaults, groups, or hosts.
job.ymlMay select the credential reference for a declared job.
--credential <name>Overrides the job or inventory-selected credential for one command.

Supported Providers

ProviderUsecreds add behavior
promptPrompt securely at runtime.No secret enrollment; reports that runtime prompting will be used.
pscredentialPowerShell module wrapper handoff only.Invalid from direct dispatch.exe; the wrapper supplies the credential in-process.
dpapi_fileLocal DPAPI-protected file.Prompts, confirms, protects with Windows DPAPI, and writes the configured file.
windows_credential_managerWindows Credential Manager target.Prompts, confirms, and writes the configured generic credential target.
azure_keyvaultAzure Key Vault secret retrieval for the endpoint password.Validates Key Vault access and secret readability; it does not store the password locally.

Config Example

dispatch:
  default_transport: psrp
  default_credential_provider: prompt

credentials:
  prod-admin:
    provider: prompt
    username: CONTOSO\prod.admin

  helpdesk-local:
    provider: dpapi_file
    username: .\helpdesk-admin
    path: C:\ProgramData\Dispatch\Credentials\helpdesk-local.cred

  domain-admin:
    provider: windows_credential_manager
    username: CONTOSO\domain.admin
    target: Dispatch/domain-admin

  kv-prod-admin:
    provider: azure_keyvault
    username: CONTOSO\prod.admin
    vault_uri: https://contoso-dispatch-kv.vault.azure.net/
    secret_name: prod-admin-password
    auth: default_azure_credential

Commands

dispatch creds list
dispatch creds add prod-admin
dispatch creds test prod-admin
dispatch creds remove prod-admin

Runtime Example

dispatch run ps .\Fix.ps1 --target PC001 --transport psrp --credential prod-admin

Azure Key Vault Auth Modes

auth controls how Dispatch authenticates to Azure Key Vault. It is not the endpoint password. The planned modes are default_azure_credential, managed_identity, and azure_cli.

Secret Rules

Passwords, tokens, SAS values, and secret values are rejected in YAML. Store only reference names, usernames, provider locations, Key Vault URIs, secret names, and auth mode names.
