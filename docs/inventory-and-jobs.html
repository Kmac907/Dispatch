Inventories And Jobs

Dispatch uses YAML for host inventories, global config, and declared jobs. Inventory files select hosts and may assign credential references. Job files describe what to run.

Inventory Example

defaults:
  transport: psrp
  credential: prod-admin

groups:
  kiosks:
    vars:
      credential: helpdesk-local
    hosts:
      - KIOSK01
      - KIOSK02

hosts:
  SERVER01:
    credential: domain-admin
  SERVER02:

Job Example

name: Fix endpoints
hosts: kiosks
transport: psrp
credential: prod-admin

tasks:
  - ps: .\Fix.ps1

Run Examples

dispatch run ps .\Fix.ps1 --inventory .\hosts.yml --target kiosks
dispatch apply .\job.yml --plan
dispatch apply .\job.yml --credential breakglass-admin

The current apply implementation plans and executes one script-first ps task. Multi-task jobs and non-ps task types remain planned v1 work.

Credential Precedence

  1. CLI --credential.
  2. job.yml credential.
  3. Host credential.
  4. Group credential.
  5. Inventory defaults credential.
  6. No credential.

Secret Rule

Inventory and job files may reference credentials by name, but must not contain passwords, tokens, SAS values, or secret material.
