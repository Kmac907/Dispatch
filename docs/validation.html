Testing And Validation

Dispatch uses automated tests plus least-invasive live validation when a slice touches endpoint execution.

Automated Suite

dotnet test .\Dispatch.sln

Live Endpoint Policy

Live endpoint validation requires one user-approved reachable Windows endpoint. Additional supplied devices that are offline or unreachable are environment availability issues, not product failures, when one approved endpoint passes.

Host Names

Keep lab host names in ignored local files such as workflow/build/test-hosts.yml, never in public docs or tracked files.

Endpoint Tests

Use least-invasive live checks, preferably stdout-only commands or temporary files under the Dispatch run folder. Do not make unrelated endpoint changes during validation.
