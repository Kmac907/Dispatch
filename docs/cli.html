Command Surface

CLI Reference

dispatch.exe is the canonical command surface. The PowerShell module wrapper maps to these commands rather than introducing a separate engine.

Command Tree

dispatch apply <job.yml>
dispatch apply <job.yml> --plan
dispatch run ps <script.ps1>
dispatch run cmd <command>
dispatch run exe <path>
dispatch push <source> --dest <remote-path>
dispatch hosts list|test|validate|graph|vars
dispatch logs list|show|tail|export|retry
dispatch creds add|list|test|remove
dispatch doctor
dispatch init job|hosts|config|all
dispatch version

Common Options

Use --target, --inventory, --transport, --credential, --config, --plan, --output, and --no-progress where supported by the command. Planned run ps script secret handoff uses --secret name=reference, separate from endpoint --credential, and plan output may render only redacted protected secret-file paths.

The current apply path supports planning, checking, and executing selected multi-task script-first ps, scalar cmd, and scalar exe jobs, plus plan/check-only copy task rendering and transport precedence across explicit CLI values, job transport, inventory transport policy, and config/default transport. Additional task execution remains planned v1 work.

Output Modes

rich is for humans. json, ndjson, yaml, and table output are for automation and stable parsing.
