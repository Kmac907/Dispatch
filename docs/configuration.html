Configuration

Dispatch uses one global machine-wide YAML config and optional command-level overrides.

Default Location

C:\ProgramData\Dispatch\config.yml

Example

dispatch:
  default_transport: psrp
  default_credential_provider: prompt

credentials:
  prod-admin:
    provider: prompt
    username: CONTOSO\prod.admin

Precedence

  1. Explicit CLI options.
  2. Explicit `--config <path>`.
  3. `C:\ProgramData\Dispatch\config.yml`.
  4. Built-in defaults.

Credential Rule

`job.yml`, `hosts.yml`, and `--credential` select a credential reference. `config.yml` defines what that reference means and how it is resolved.

Security

Do not store passwords, tokens, SAS values, or secret material in YAML. Only store references, usernames, provider locations, Key Vault URIs, and secret names.
