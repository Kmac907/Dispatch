Config

Configuration

Dispatch uses one global machine-wide YAML config and optional command-level overrides. This config is the single catalog for Dispatch defaults and credential provider metadata.

Default Location

C:\ProgramData\Dispatch\config.yml

Example

dispatch:
  default_transport: psrp
  default_credential_provider: prompt

credentials:
  prod-admin:
    provider: prompt
    username: CONTOSO\prod.admin

Config Loading

  1. Explicit --config <path>.
  2. C:\ProgramData\Dispatch\config.yml.
  3. Built-in defaults.

Runtime Precedence

Explicit CLI options override job, inventory, global config, and built-in defaults for the current command.

Credential Catalog

job.yml, hosts.yml, and --credential only select a credential reference. The loaded global config defines what that reference means and how it is resolved.

Security

Do not store passwords, tokens, SAS values, or secret material in YAML. Only store references, usernames, provider locations, Key Vault URIs, and secret names.
